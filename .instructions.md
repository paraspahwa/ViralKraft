Repository-wide testing and verification standards:

1. Reproduce before fix:
- Confirm expected vs actual behavior with a concrete failing path (error, failing check, or reproducible steps).

2. Validate narrowly, then broadly:
- Run the smallest relevant check first (targeted file/test/endpoint).
- If the narrow check passes, run broader validation (typecheck/build/tests) when available.

3. Verify changed risk surfaces:
- For API/payment/auth/webhook changes, test success and failure paths, including idempotency and error handling.
- For UI changes, verify affected routes/components and ensure production build still succeeds.

4. Keep fixes minimal and safe:
- Prefer smallest diff that resolves the issue.
- Avoid unrelated refactors, dependency churn, or formatting-only edits in bug/perf tasks.

5. Report evidence in outcomes:
- List what was validated, what passed/failed, and what was not run.
- Call out residual risks and follow-up checks if coverage is incomplete.
